# 穿越火线模拟器 数据库字段说明（SQLite）

文档版本：2026-02-27  
数据库文件：`lottery_simulator/data/lottery.db`

## 1. 总览

本项目当前使用 8 张表：

1. `purchase_options`：购买钥匙档位配置。
2. `points_groups`：积分组（用于“积分池”卡片）配置。
3. `prize_items`：奖池与兑换道具主表（最核心）。
4. `pool_layout_settings`：奖池展示布局配置。
5. `pool_palette_priority`：卡片颜色优先级。
6. `popup_highlight_rules`：弹窗高亮规则。
7. `system_settings`：全局系统配置（推荐放“改一次全局生效”的项）。
8. `schema_migrations`：数据库迁移记录。

当前配置值（已落库）：

- `system_settings.points_image_url = /cf_images/jifen.png`
- `system_settings.redeem_destination = warehouse`
- `pool_layout_settings.ring_rows = 3`

## 2. 表结构详细说明

## 2.1 `purchase_options`（购买配置）

用途：控制“充值/买钥匙”按钮对应的价格与钥匙数量。

字段：

- `option_key`（TEXT, PK）：档位唯一键。示例：`single`、`bundle`。
- `label`（TEXT, NOT NULL）：前端展示文案。示例：`10元1抽`。
- `price`（INTEGER, NOT NULL）：购买价格（元）。
- `keys_count`（INTEGER, NOT NULL）：该档位赠送钥匙数。

影响功能：

- `/api/config` 返回购买选项。
- `/api/buy` 根据 `option_key` 扣费并增加钥匙。
- 模拟训练 `/api/simulate` 用它计算成本策略。

---

## 2.2 `points_groups`（积分组）

用途：定义“积分组卡片”，该组下挂若干 `points_child` 子项（在 `prize_items` 表中）。

字段：

- `group_key`（TEXT, PK）：积分组唯一键。常见：`points_pool`。
- `name`（TEXT, NOT NULL）：组名称，前端展示。
- `image_url`（TEXT, NOT NULL, default ''）：组图片地址。
- `card_bg_color`（TEXT, NOT NULL, default ''）：卡片背景样式（可填 CSS 渐变）。
- `sort_order`（INTEGER, NOT NULL, default 100）：排序，越小越靠前。
- `palette_key`（TEXT, NOT NULL, default 'gray'）：颜色主题键。

影响功能：

- 奖池中的 `points_group` 行。
- 前端奖池卡片样式与排序。

---

## 2.3 `prize_items`（奖池/兑换主表）

用途：全项目最核心表。既承载抽奖概率，也承载兑换、分解、图片、样式、限兑规则。

字段：

- `item_id`（TEXT, PK）：道具唯一 ID。
- `name`（TEXT, NOT NULL）：道具名/积分项名。
- `item_type`（TEXT, NOT NULL）：类型，取值只能是：
  - `item`：实体道具。
  - `points_child`：积分子项（挂在某个 `group_key` 下）。
- `in_pool`（INTEGER, NOT NULL, default 1）：是否参与抽奖池。`1`=参与，`0`=不参与。
- `pool_weight`（REAL, NOT NULL, default 0）：抽奖权重（不是百分比）。
- `group_key`（TEXT, 可空）：当 `item_type='points_child'` 时应指向 `points_groups.group_key`。
- `points_value`（INTEGER, NOT NULL, default 0）：积分数值。仅 `points_child` 有意义。
- `exchange_points`（INTEGER, NOT NULL, default 0）：兑换所需积分。`>0` 表示可兑换。
- `decompose_points`（INTEGER, NOT NULL, default 0）：分解获得积分。
- `decompose_keys`（INTEGER, NOT NULL, default 0）：分解获得钥匙。
- `direct_to_warehouse`（INTEGER, NOT NULL, default 0）：抽中后去向。`1`=直接入仓库，`0`=进暂存箱。
- `image_url`（TEXT, NOT NULL, default ''）：常规图片地址。
- `card_bg_color`（TEXT, NOT NULL, default ''）：卡片背景样式。
- `sort_order`（INTEGER, NOT NULL, default 100）：排序。
- `popup_image_url`（TEXT, NOT NULL, default ''）：开奖弹窗专用图。
- `palette_key`（TEXT, NOT NULL, default 'orange'）：颜色主题键。
- `redeem_limit_enabled`（INTEGER, NOT NULL, default 0）：是否启用限兑。
- `redeem_limit_count`（INTEGER, NOT NULL, default 0）：限兑数量上限。
- `redeem_tag_left`（TEXT, NOT NULL, default ''）：兑换卡左上角标签文本。
- `redeem_tag_right`（TEXT, NOT NULL, default '不可交易'）：兑换卡右上角标签文本。

关键业务规则：

- 抽奖池只读取 `in_pool=1` 的记录。
- 概率由同层级 `pool_weight` 按比例自动计算。
- `exchange_points>0` 的 `item` 会进入兑换商城。
- 兑换是否进仓库/暂存箱，不再写死，现由 `system_settings.redeem_destination` 控制。
- 前端“道具分解表/模拟模块”会按活动分解模式动态展示：积分活动读 `decompose_points`，钥匙活动读 `decompose_keys`。

---

## 2.4 `pool_layout_settings`（奖池布局）

用途：控制前端奖池环形布局参数。

字段：

- `setting_key`（TEXT, PK）：配置项名。
- `setting_value`（TEXT, NOT NULL）：配置值字符串。

当前已用配置：

- `ring_rows`：环形行数，后端会做最小值保护（至少 3）。

---

## 2.5 `pool_palette_priority`（颜色优先级）

用途：同一批道具在展示时，不同 `palette_key` 的排序优先级。

字段：

- `palette_key`（TEXT, PK）：颜色键。
- `priority`（INTEGER, NOT NULL）：优先级，越小越靠前。

默认示例：`orange=1`、`purple=2`、`blue=3`、`gray=4`。
当前活动：`red=1`、`orange=2`、`purple=3`、`blue=4`、`gray=5`。

---

## 2.6 `popup_highlight_rules`（弹窗高亮规则）

用途：决定哪些颜色等级会触发“稀有弹窗/高亮队列”。

字段：

- `palette_key`（TEXT, PK）：颜色键。
- `enabled`（INTEGER, NOT NULL, default 0）：`1` 启用，`0` 关闭。

默认示例：橙/紫/蓝开启，灰色关闭。
当前活动：红/橙/紫/蓝开启，灰色关闭。

---

## 2.7 `system_settings`（全局系统配置）

用途：放“只改数据库就全局生效”的系统级设置。

字段：

- `setting_key`（TEXT, PK）：配置项名。
- `setting_value`（TEXT, NOT NULL）：配置值。

当前已实现配置：

- `points_image_url`：积分区域图片地址（`/api/config` 直接读取）。
- `redeem_destination`：兑换道具发放目标：
  - `warehouse`：直接进仓库。
  - `stash`：进入暂存箱并生成待处理记录。

建议后续也放进这里的项：

- 默认弹窗开关。
- 历史记录保留条数。
- 特殊活动文案。

---

## 2.8 `schema_migrations`（迁移记录）

用途：记录“只执行一次”的迁移，避免每次启动覆盖业务配置。

字段：

- `migration_key`（TEXT, PK）：迁移唯一标识。
- `applied_at`（INTEGER, NOT NULL）：执行时间戳（Unix 秒）。

当前示例迁移：`redeem_limit_default_v1`。

## 3. 表关系与数据流

主要关系：

- `points_groups.group_key` 1 对多 `prize_items.group_key`（仅 `points_child` 使用）。

抽奖数据流：

1. 从 `prize_items` 读取 `in_pool=1` 的 `item` 和 `points_child`。
2. 若为 `points_child`，按 `group_key` 聚合到 `points_groups`。
3. 依据 `pool_weight` 计算概率。
4. 抽中 `item` 后按 `direct_to_warehouse` 入仓库或暂存箱。

兑换数据流：

1. 从 `prize_items(exchange_points>0)` 进入兑换列表。
2. `redeem_limit_enabled + redeem_limit_count` 校验限兑。
3. 发放目标由 `system_settings.redeem_destination` 决定。

## 4. 常用 SQL 修改示例（改库即生效）

1. 修改兑换去向为暂存箱：

```sql
UPDATE system_settings
SET setting_value='stash'
WHERE setting_key='redeem_destination';
```

2. 修改积分图：

```sql
UPDATE system_settings
SET setting_value='/cf_images/jifen-1.png'
WHERE setting_key='points_image_url';
```

3. 下架某个道具（不参与抽奖）：

```sql
UPDATE prize_items
SET in_pool=0
WHERE item_id='longxiao_sound_card';
```

4. 提高某道具命中概率（提高权重）：

```sql
UPDATE prize_items
SET pool_weight=1.2
WHERE item_id='legend_haojie';
```

5. 开启某道具限兑 1：

```sql
UPDATE prize_items
SET redeem_limit_enabled=1,
    redeem_limit_count=1,
    redeem_tag_left='单大区限兑1'
WHERE item_id='legend_elisha';
```

6. 新增购买档位：

```sql
INSERT INTO purchase_options(option_key,label,price,keys_count)
VALUES('mega','300元35抽',300,35);
```

## 5. 注意事项

1. `pool_weight` 是相对权重，不是百分比。
2. `item_type='points_child'` 时应配置 `group_key` 与 `points_value`。
3. `exchange_points>0` 才会显示在兑换列表。
4. `redeem_destination` 建议只用 `warehouse` 或 `stash`。
5. 数据库改动通常刷新页面即可生效；若服务端有缓存逻辑再考虑重启。
